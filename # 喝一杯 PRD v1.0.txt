# 喝一杯 PRD v1.0
## 1. 产品定位
一句话：一款基于情绪推荐的**消费 + 线下社交**产品。

## 2. 目标用户与核心场景
### 主用户画像
一二线城市年轻人。

### 三个典型场景
1. **独酌型 → 留言型**
周三晚上加班完想喝一杯，打开 App 输入「有点累但不想睡」，拿到 3 杯推荐，选最近的店，看到留言板共鸣内容，到店并留言，无主动社交但获得情绪陪伴。
2. **想见人型 → 发车**
周五晚上想找陌生人喝酒，输入「想被陌生人逗笑」，选酒并在店铺发车，设定人数与时间，15 分钟内凑齐车队，AI 带破冰，线下相聚。
3. **目的型 → 模糊搜索**
记不清酒吧全名，首页搜索 → 找店结果页 → 查看留言板氛围 → 决定是否到店。

## 3. 信息架构与核心链路
```
首页(三入口)
├── A. 输入心情
├── B. 输入口味 → 推荐 3 杯酒 → 选 1 杯 → 酒吧列表(按距离) → 选 1 家 → 酒吧详情页
└── C. 输入酒吧名 → 找店结果列表 → 酒吧详情页

酒吧详情页(沉浸式留言板 + 悬浮车队入口)
留言板(默认主视图)
├── 浏览 24h 内留言
└── 发布留言(预填心情 + 预挂那杯酒,可编辑/取消)
车队(右下角悬浮按钮 → 半屏抽屉)
├── 发车
├── 上车(申请制)
└── AI 匹配(由 Agent 主动私邀) → 成团 → 进入车队群 → 机器人主持(签到/破冰/游戏) → 线下到店核销
```

## 4. 详细功能需求
### 4.1 首页
- **布局**：一个输入框，无 Tab、无下拉、无类型选择。
- **Placeholder**：随机轮播，示例：「聊聊心情、想喝点什么、或者直接说出某家店的名字」。
- **意图识别逻辑**：
  - 找店：含已知店铺名/模糊匹配 → 酒吧消歧列表
  - 口味：含明确口味词 → 推荐 3 杯
  - 心情：非以上两类 → 推荐 3 杯
- **边界情况处理**：
  - 混合意图：同时命中心情+口味 → 走推荐 3 杯
  - 歧义词汇（如蓝调）：优先找店，底部提供回退入口
  - 极短输入（1-2 字）：直接进入心情推荐
  - 无法识别/空输入：轻提示，不做强校验
- **交互细节**：
  - 输入内容前端持久化，用于留言板预填
  - 识别结果不弹窗确认，直接跳转，提供回退入口
  - 输入框下方展示最近输入/热门心情词

### 4.2 推荐 3 杯酒（心情/口味入口）
- **输入**：用户自然语言文本（心情/口味）
- **输出**：3 杯酒，画廊式排序，附推荐语
- **推荐底层**：
  1. NLP 解析 → 提取情绪/口味向量
  2. 映射塔罗大阿卡纳（22 张）作为中介符号
  3. 符号匹配酒款向量库，输出 Top3
  4. 生成 15-25 字塔罗式推荐语
- **卡片字段**：酒款图、酒名、推荐语、价格区间
- **点击行为**：进入卖该酒的酒吧列表

### 4.3 酒吧列表页（由酒款进入）
- **排序**：默认距离优先，支持价格/评分切换
- **卡片字段**：店铺主图、名称、距离、该酒价格、留言数/车队数
- **点击**：进入店铺详情页，保留所选酒状态

### 4.4 找店结果列表（模糊搜索入口）
- **用途**：店铺名模糊匹配二次确认
- **卡片字段**：店铺主图、完整店名、地址、距离、留言数/车队数
- **点击**：直接进店铺详情页，跳过推荐环节
- **特殊处理**：无预挂酒，留言时可选店内酒款

### 4.5 酒吧详情页（核心页面）
- **视觉层级**：
  1. 顶部店铺信息条（可上滑收起）
  2. 留言板主视图（沉浸式）
  3. 右下角悬浮车队入口
- **设计原因**：留言板为情绪核心，悬浮按钮降低社交门槛，保持克制感

### 4.6 留言板
- **发布权限**：所有访问详情页用户，推荐链路用户带专属标识
- **身份**：昵称/匿名（匿名显示「路过的客人」+塔罗图标）
- **发布交互**：首页心情预填 → 预挂所选酒 → 一键发布
- **卡片字段**：用户身份、文本、挂载酒、时间
- **规则**：24h 后自动隐藏；v1.0 无点赞/评论/私信
- **社交约束**：仅通过车队社交，避免泛陌生人交友

### 4.7 车队（发车/上车/AI 匹配）
#### 4.7.1 发车
- **字段**：集合时间、总人数、性别约束、集合地点、招募语、想喝什么
- **状态**：发起后进入上车列表，开局前 30 分钟内取消计信用

#### 4.7.2 上车
- **列表**：按开局时间最近排序，展示发起人、时间、剩余位置等
- **机制**：申请制，30 分钟发起人未操作默认通过
- **退出**：开局前 1 小时以上自由退出，否则计信用

#### 4.7.3 AI 匹配
- **入口**：AI 匹配 Tab
- **输入**：留言、所选酒、时间/性别偏好
- **动作**：Agent 私邀，全员接受则成团

### 4.8 车队群与 AI Agent 主持人
- **生命周期**：成团至当日 24:00/开局后 4 小时，记录保留 7 天
- **Agent 能力**：
  1. 签到（到店核销依据）
  2. 动态破冰话题
  3. 线下游戏发牌
  4. 氛围维护
- **立场**：推动线下见面，不滞留线上聊天

## 5. 推荐与匹配逻辑（框架级）
### 5.1 西方符号体系
- v1.0 主用**塔罗大阿卡纳 22 张**作情绪-酒款中介
- 星座留作 v1.1 用户标签
- 文案克制，不深度使用塔罗术语

### 5.2 情绪向量复用
推荐酒款与 AI 匹配车队共用同一套情绪向量，形成数据飞轮。

### 5.3 算法待定项
- 塔罗 22 牌 ↔ 酒款向量映射表
- 酒型 MBTI 16 型设计
- 留言文本 Embedding 方案
- 冷启动兜底匹配规则

## 6. 商业模式
### 6.1 收入来源（三层）
#### 第一层：店家侧（主收入 60%+）
- 车队成团抽佣：人头 5-10 元
- 店家专属车队套餐：抽佣 15-20%
- 入驻认证费：辅助收入
- 氛围数据服务：长线

#### 第二层：用户侧（增值 25%）
- 塔罗深度报告：付费解锁 30 天饮酒情绪报告

#### 第三层：品牌合作（补充 15%）
- 洋酒联名特调
- 线下活动赞助

### 6.2 核销机制
三选二判定真到店：
1. 2/3 以上成员到店签到
2. 破冰有效互动 >5 条
3. 晚间群活跃

### 6.3 冷启动建议
上线 3 个月内不收用户费用，试点车队套餐，核心 KPI：单店日均成团数、真到店率。
